Basics

Git workflows can be divided into these main categories:

  • Recording - save the changes in your repository.
  • Updating - fetch down all new information from a remote repository and store it.
  • Sharing - store your changes in a remote repository.

If you don't have access to any git repository just create a fresh one to play around with. Create and open up a new directory:

mkdir cats
cd cats

Then initialize a new git repository with:

git init

As simple as that!

Recording

You already made some changes to a webpage in this git repository. Typing git status will always show you the current status of your working directory:

git status

Git knows that index.html is changed and it's ready to save your modifications.

To do that, you have to create a commit. Putting together a git commit is like shopping where the commit is your shopping cart. You carefully look at the available items and decide what goes into your shopping cart and what doesn't.

Use git add to put a modified file into your commit:

git add index.html

If you don't want to add anything else, record the selected modifications:

git commit -m "awesome text"

Commits are created along with a log message which describes the changes made in that commit.

Update

git pull <remote> <branch>

This is one of the concepts git uses to distribute content. It will incorporate the changes made by others into your local repository. To make your repository up to date with the remote repository, use: 

git pull

When <remote> isn't specified, origin is used. So the equivalent of the above command would be:

git pull origin

Specifing <branch> will pull the changes from that branch into your current branch. To pull the changes from branch awesome-feature-23 to your local branch do:

git pull origin awesome-feature-23

Common errors

From /a/remote/repository
  * branch            master     -> FETCH_HEAD
error: Untracked working tree file 'myproject/example.js' would be overwritten by merge.  Aborting

Another error message caused by the same problem: 

From /a/remote/repository
  * branch            master     -> FETCH_HEAD
error: Your local changes to 'foo.txt' would be overwritten by merge. Aborting. Please, commit your changes or stash them before you can merge.

Both errors indicate that you have some unsaved changes that would be overwritten by the modifications made by others to the same file.

As a solution, commit your changes, then repeat the pull.

Share

Like other contributors of the repository, you also want to share your changes with others. Use git push to update the remote repository with your changes. 

git push origin

Merge

git merge new-feature

Undo

git reset HEAD~1
